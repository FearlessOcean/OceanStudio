# 星护（XingHu）代码说明文档

## 运行环境

| 项目 | 要求 |
|------|------|
| 硬件 | 鲁班猫4（RK3588S · 6 TOPS NPU · 4 GB LPDDR4X） |
| 操作系统 | Ubuntu 22.04 Server ARM64 |
| Python | 3.12 |
| 包管理 | venv（板子端）/ uv（PC 开发端） |

```

│   └── alerts.log            # 告警历史日志

    ├── agent.py         # SmartHomeAgent 核心：三通道调度、RAG注入、历史管理
```

## 安装与运行

### NPU 模式（鲁班猫4）

```bash
# 安装依赖
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 启动（NPU 推理，需提前放置 .rkllm 模型文件）
XINGHU_USE_NPU=1 python main.py
```

### Ollama 仿真模式（PC 开发调试）

```bash
# 需要本地运行 Ollama 并拉取模型
ollama pull qwen2.5:1.5b

python main.py
```

启动后选择：
- `1` 自动运行 6 个预设演示场景并输出性能报告
- `2` 进入交互式多轮对话

### 运行测试套件

```bash
python test_runner.py   # 运行 58 个标准测试用例，输出 test_results.json
```

## 模型转换（x86 Ubuntu → RK3588S .rkllm）

NPU 模式需要将 Hugging Face 原始模型转换为 `.rkllm` 格式，**必须在 x86 Linux 环境下完成**（推荐 WSL2 Ubuntu 22.04）。

### 1. 安装 rkllm-toolkit

rkllm-toolkit **不在 PyPI**，需从瑞芯微官方渠道获取 wheel 包后手动安装：

```bash
# 建议使用 Python 3.10 / 3.11（toolkit 对版本有要求）
pip install rkllm_toolkit-*.whl
```

> 官方仓库：https://github.com/airockchip/rknn-llm

### 2. 下载原始模型

```bash
# 需要 huggingface-cli 或 modelscope
pip install huggingface_hub
huggingface-cli download Qwen/Qwen2.5-1.5B-Instruct --local-dir ./Qwen2.5-1.5B-Instruct
```

### 3. 运行转换脚本

新建 `convert.py`，内容如下：

```python
from rkllm.api import RKLLM

llm = RKLLM()

# 以 float16 精度加载原始权重
llm.load_huggingface(
    model='./Qwen2.5-1.5B-Instruct',
    dtype='float16'
)

# W8A8 量化，目标平台 RK3588
llm.build(
    do_quantization=True,
    optimization_level=1,
    quantized_dtype='w8a8',
    target_platform='RK3588'
)

# 导出 .rkllm 文件（约 1.9 GB，耗时约 7 分钟）
llm.export_rkllm('./qwen2_5-1_5b-w8a8.rkllm')
```

```bash
python convert.py
```

### 4. 部署到板子

```bash
# 将 .rkllm 文件传到鲁班猫4（与代码默认路径保持一致）
scp qwen2_5-1_5b-w8a8.rkllm user@<板子IP>:/home/cat/models/qwen2.5-1.5b-w8a8-rk3588.rkllm

# 将 librkllmrt.so 放到系统库路径（随 rkllm-runtime 包提供）
scp librkllmrt.so user@<板子IP>:/usr/local/lib/
ssh user@<板子IP> "sudo ldconfig"
```

若模型放在其他路径，通过环境变量指定：

```bash
XINGHU_MODEL_PATH=/your/path/model.rkllm XINGHU_USE_NPU=1 python main.py
```

### 5. 部署嵌入模型（语义 RAG）

系统使用 `BAAI/bge-small-zh-v1.5`（约 90 MB）进行语义检索。板子**首次运行**时会自动从 HuggingFace 下载。

**（自动下载）**：首次启动时自动完成，无需额外操作。

**（手动传输）**：

```bash
# PC 端下载模型
pip install huggingface_hub
huggingface-cli download BAAI/bge-small-zh-v1.5 --local-dir ./bge-small-zh-v1.5

# 传到板子的 HuggingFace 缓存目录
scp -r ./bge-small-zh-v1.5 user@<板子IP>:~/.cache/huggingface/hub/models--BAAI--bge-small-zh-v1.5/
```

> 若嵌入模型加载失败，系统会静默降级为关键词检索，功能不中断。

### 6. 启动 NPU 模式

```bash
XINGHU_USE_NPU=1 python main.py
```

## Streamlit 监控面板

监控面板运行在 Windows PC 端，通过 SSH 连接板子采集实时指标。

```bash
# 板子端：启动性能指标采集服务（后台运行）
python board_monitor.py &

# PC 端：安装依赖并启动（Windows/Linux 均可）
pip install streamlit paramiko
streamlit run dashboard.py
```

> `streamlit` 和 `paramiko` 仅 PC 端需要，不在板子的 `requirements.txt` 中。

浏览器访问 `http://localhost:8501`，可实时查看 CPU/内存/温度、设备状态、告警记录。

## systemd 开机自启（生产部署）

```bash
# 在板子上创建服务文件
sudo nano /etc/systemd/system/xinghu.service
```

```ini
[Unit]
Description=XingHu Smart Home Agent
After=network.target

[Service]
User=root
WorkingDirectory=/root/xinghu
Environment=XINGHU_USE_NPU=1
ExecStart=/root/xinghu/.venv/bin/python3 main.py
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable xinghu
sudo systemctl start xinghu
sudo systemctl status xinghu
```

## 环境变量

| 变量 | 值 | 说明 |
|------|----|------|
| `XINGHU_USE_NPU` | `1` | 启用 RK3588S NPU 推理模式 |
| `XINGHU_MODEL_PATH` | 路径字符串 | 覆盖默认模型路径（默认：`/home/cat/models/qwen2.5-1.5b-w8a8-rk3588.rkllm`） |
| 不设置 | — | 使用 Ollama 仿真模式（开发调试） |

## 核心模块说明

| 模块 | 文件 | 职责 |
|------|------|------|
| SmartHomeAgent | `src/agent.py` | 意图预检、RAG 注入、三通道工具调度、历史压缩 |
| 工具函数 | `src/tools.py` | 设备控制、健康记录/查询、告警、用药提醒、传感器 |
| 知识库 RAG | `src/knowledge_base.py` | bge-small-zh-v1.5 语义向量检索，支持近义词召回 |
| NPU 推理后端 | `src/llm_rknn.py` | ctypes 封装 librkllmrt.so，LangChain 兼容接口 |
| 后台守护进程 | `src/monitor.py` | 传感器巡检（30s）、用药提醒调度（60s） |
| 性能记录 | `src/profiler.py` | 端到端延迟、内存增量、工具调用次数统计 |
